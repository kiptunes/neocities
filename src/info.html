About the Site

I made this site to use as a css without js sandbox ! It uses the Bulma CSS framework (specifically the alt version without helpers) for theming, responsiveness, and other components like the breadcrumbs at the bottom of this page. Since it doesn't come with js, but has components like dropdowns that assume exterior js implementation to work, I recoded dropdowns on my own to work with only css.

FAQ

How long did this take you?

According to Wakatime, as of now, 15 hours!

Why not use javascript?

This site doesn't have to be lightweight, and most browsers support javascript by now. However, I'd spent some time away from web development at the time of building this site and wanted to regain familiarity with css specifically, so I decided to try and not use any javascript at all on this site. At some point in the future, I might overhaul the site to implement js, since a lot of things are held together with duct tape. For now, though, if it works, it works.

Why use a framework?

Responsiveness (the ability of a website to adapt to different screen sizes) is important to me, and it's just a little bit faster to use a framework to accomplish that. I guess a lot of the bulma css code is redundant, though, since I didn't use a lot of it.

Can I copy or modify this site's code?

Yes! My code is free use without attribution, but if you use anything from the source code that requires Bulma CSS, keep in mind that it uses the MIT license, which has different requirements.
